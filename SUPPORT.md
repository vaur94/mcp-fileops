# Support

> English | [Turkce](./SUPPORT.tr.md)

Use the repository's public GitHub surfaces for normal product and documentation support.

## Where to ask for help

- open a GitHub issue for reproducible bugs or documentation problems
- open a GitHub discussion only if the repository enables discussions later
- use pull requests for concrete documentation or code improvements

## What to include

- operating system and Node.js version
- npm version
- MCP client name if relevant
- the command, config, or workflow you used
- the exact error output when possible

## Security issues

Do not report exploitable security findings through normal public issues. Use the private reporting guidance in `SECURITY.md` instead.

## Scope note

This repository does not currently document a separate commercial support channel or SLA.
